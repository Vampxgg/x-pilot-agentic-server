# 图片文字识别（OCR）

## 目标

从用户给出的图片中还原可读文本，尽量保持段落与列表结构；对公式、代码、表格单独说明格式。

## 首选做法（本环境）

1. 取得图片的**可读路径**（工作区相对路径或绝对路径）。若用户只贴了图，请让用户保存为文件并给出路径，或确认附件在对话中已可被读取。
2. 使用 **Read** 工具读取该图片文件（支持常见位图格式）。
3. 根据图像内容输出：
   - **正文**：按阅读顺序输出，必要时用标题层级组织。
   - **不确定处**：用 `[?]` 标出无法确认的字符或词。
   - **代码/命令**：放在 Markdown 代码块中，并注明语言（若可判断）。
   - **表格**：用 Markdown 表格或对齐的纯文本表；复杂表可说明“列/行对应关系”。
   - **多语言**：按图中实际语言转写；混合语言分段输出。

## 质量约束

- 不臆造图中不存在的文字；看不清就标 `[?]` 或说明“该区域无法辨认”。
- 保留数字、单位、标点、大小写；专有名词保持原样。
- 若图为 UI：可简述控件层级（可选），但**以可见文字为主**。

## 何时改用本地 OCR 引擎

仅在用户明确要求**可复现的批量 OCR**、或视觉模型反复读不准时，再建议在用户机器上安装 **Tesseract** 并用 `pytesseract`/`tesseract` CLI 处理；本技能默认不依赖额外安装。

## 反模式

- 未读取图片文件就凭文件名或猜测输出“识别结果”。
- 把模糊笔画强行补成“看起来像”的词而不标注不确定。
