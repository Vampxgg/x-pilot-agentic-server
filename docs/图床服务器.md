# 图床服务（Image Hub）接口文档

本文档描述 X-Pilot Agentic Server 内置的全局图片库 HTTP API。实现代码：`src/api/routes/image.routes.ts`、核心逻辑：`src/services/image-service.ts`。

---

## 1. 概述

### 1.1 能力

- 全局共享图片池：文件落盘于 `data/images/library/{sha256}.{ext}`，元数据索引 `data/images/metadata.json`。
- 按内容 SHA-256 去重：相同二进制只存一份；多条元数据若引用同一 `hash` 会共享物理文件。
- 访问 URL：由 `storagePath` 与 `IMAGE_SERVE_BASE_URL` 动态拼接（见下文「访问 URL 规则」）。
- **Resolve**：根据章节标题与关键词，与元数据中 `sourceDetail.imgKeywords` 做交集打分匹配（Phase 1 算法）。

### 1.2 与会话级图片的关系

- 工作区会话图片仍写入：`data/tenants/{tenantId}/users/{userId}/workspaces/{sessionId}/assets/images/`（`image_generate` 等行为不变）。
- 全局图床与会话目录独立；`image_generate` 在保存到会话目录后会**异步**尝试将同一份文件再入库到全局池（失败不阻塞生成）。

### 1.3 认证

除 `/api/health` 与静态文件前缀 `/api/files/` 外，所有请求经 `authMiddleware`：

| 场景 | 行为 |
|------|------|
| 未配置 `config/tenants.yaml` | `tenantId` / `userId` 固定为 `default` |
| 已配置租户 | 需 `Authorization: Bearer <apiKey>` 或查询参数 `apiKey` |
| `NODE_ENV=production` 且无 API Key | `401 Missing API key` |

写入类接口（上传、from-url、resolve 命中后内部会更新 `usageCount`）使用解析得到的 `tenantId`、`userId` 写入元数据。

### 1.4 访问 URL 规则

服务端计算字段 `url`（不持久化在元数据中）：

- 若设置环境变量 `IMAGE_SERVE_BASE_URL`（末尾不要多余 `/`）：  
  `url = {IMAGE_SERVE_BASE_URL}/{storagePath}`
- 否则：  
  `url = /api/files/{storagePath}`  

其中 `storagePath` 形如 `images/library/{hash}.jpg`，与 `fastifyStatic` 挂载的 `data/` 根目录一致。

## 2. 数据模型

### 2.1 ImageMetadata（元数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string (UUID) | 记录主键 |
| `hash` | string | 文件内容 SHA-256 |
| `tenantId` | string | 入库时租户 |
| `userId` | string | 入库时用户 |
| `sessionId` | string? | 可选，会话 ID |
| `filename` | string | 原始/逻辑文件名 |
| `mimeType` | string | 如 `image/jpeg` |
| `size` | number | 字节数 |
| `width` / `height` | number? | 可选，当前实现可能未填 |
| `source` | enum | `upload` \| `ai_generated` \| `web_import` \| `knowledge_recall` |
| `sourceDetail` | object? | 溯源与检索辅助字段 |
| `sourceDetail.originalUrl` | string? | 外部/知识库原始 URL |
| `sourceDetail.imgKeywords` | string[]? | **Resolve 核心**：知识库 `img_keywords` 拆成的词列表 |
| `sourceDetail.imgDescription` | string? | 知识库图片描述 |
| `sourceDetail.segmentId` / `documentId` / `knowledgeBaseId` | string? | 可选溯源 |
| `sourceDetail.generationPrompt` / `generationModel` | string? | AI 生成来源 |
| `tags` | string[] | 标签 |
| `domain` | string? | 领域 |
| `description` | string? | 描述 |
| `storagePath` | string | 相对 `data/` 的路径 |
| `usageCount` | number | 引用次数（resolve 命中会自增） |
| `lastUsedAt` | string? | ISO 时间 |
| `visibility` | enum | `shared`（默认）\| `tenant` \| `private`（Phase 1 列表/过滤以 `shared` 为主） |
| `extra` | object? | 扩展字段 |
| `createdAt` / `updatedAt` | string | ISO 时间 |

响应中常附带计算字段 **`url`**（见 1.4）。

### 2.2 Resolve 匹配逻辑（Phase 1）

1. 将 `sectionTitle` 与 `keywords` 拼接后按标点/空白分词，**仅保留长度 ≥ 2 的 token**。
2. 对每条元数据，取 `sourceDetail.imgKeywords`（无则视为空数组），与输入 token 做**模糊交集**：任一 imgKeyword 与任一 input token 存在互相 `includes` 则计 1 分（每个 imgKeyword 最多贡献 1 分）。
3. **得分 ≥ 2** 的记录进入候选；按分数降序，其次 `usageCount` 降序。
4. 取前 `imageCount` 条作为 `resolved`，并对这些记录执行 `usageCount++`、`lastUsedAt` 更新。
5. `unresolved = max(0, imageCount - resolved.length)`。

**注意**：无 `imgKeywords` 的图片（如仅 AI 生成、未带关键词的导入）在 Resolve 中通常无法命中，需依赖后续 Phase 扩展（标签/描述/向量等）。

## 3. HTTP API 一览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/images/upload` | multipart 上传入库 |
| GET | `/api/images` | 条件列表 |
| GET | `/api/images/:id` | 元数据 |
| GET | `/api/images/:id/file` | 二进制流 |
| PATCH | `/api/images/:id` | 更新元数据 |
| DELETE | `/api/images/:id` | 删除记录（无其它同 hash 记录时删文件） |
| POST | `/api/images/from-url` | 拉取 URL 入库 |
| POST | `/api/images/resolve` | 按章节上下文匹配图片 |
| POST | `/api/images/fetch` | **一站式图片获取**（四级管线：图床匹配→KB收割→联网搜图→AI生成） |

以下 `:id` 均为元数据中的 UUID（`ImageMetadata.id`），**不是**文件 hash。

## 4. 接口详情

### 4.1 `POST /api/images/upload`

**Content-Type**: `multipart/form-data`

**表单**

- **文件**：任意字段名均可；服务端取 `request.file()` 的**第一个**文件。
- **Query（可选）**
  - `tags`：逗号分隔标签，如 `教育,示意图`
  - `domain`：领域字符串
  - `description`：描述字符串

**行为**

- `source` 固定为 `upload`
- `visibility` 默认 `shared`（由服务层默认）
- `tenantId` / `userId` 来自认证中间件

**响应** `201`

- Body：完整 `ImageMetadata` + `url`

**错误**

- `400`：`{ "error": "No file uploaded" }`

---

### 4.2 `GET /api/images`

**Query（均可选）**

| 参数 | 说明 |
|------|------|
| `tenantId` | 传入时：保留 `tenantId` 等于该值 **或** `visibility === "shared"` 的记录 |
| `userId` | 精确匹配 `userId` |
| `source` | `upload` / `ai_generated` / `web_import` / `knowledge_recall` |
| `domain` | 精确匹配 `domain` |
| `tags` | 逗号分隔；记录需 **至少包含其中一个** tag |
| `limit` | 默认 `100` |
| `offset` | 默认 `0` |

**行为**

- 过滤后做数组切片 `[offset, offset + limit)`。
- **注意**：响应里的 `total` 当前为**本页条数**（`items.length`），不是全库总数。

**响应** `200`

```json
{
  "items": [ { "...ImageMetadata": "...", "url": "https://..." } ],
  "total": 10
}
```

---

### 4.3 `GET /api/images/:id`

**响应** `200`：元数据 + `url`

**错误** `404`：`{ "error": "Image not found" }`

---

### 4.4 `GET /api/images/:id/file`

**响应** `200`

- `Content-Type`: 记录中的 `mimeType`
- `Cache-Control`: `public, max-age=31536000, immutable`
- Body：图片文件流

**错误** `404`：文件不存在或元数据不存在

---

### 4.5 `PATCH /api/images/:id`

**Content-Type**: `application/json`

**Body（字段均可选，仅发送需要更新的字段）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `tags` | string[] | 标签 |
| `domain` | string | 领域 |
| `description` | string | 描述 |
| `visibility` | string | `shared` \| `tenant` \| `private` |
| `extra` | object | 扩展 JSON |

**响应** `200`：更新后的记录 + `url`

**错误** `404`

---

### 4.6 `DELETE /api/images/:id`

删除元数据记录；若没有其他记录共用同一 `hash`，则删除磁盘文件。

**响应** `200`：`{ "success": true }`

**错误** `404`

---

### 4.7 `POST /api/images/from-url`

**Content-Type**: `application/json`

**Body**

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `url` | 是 | string | 可访问的图片 URL |
| `sessionId` | 否 | string | 会话 |
| `source` | 否 | string | 默认 `web_import`；知识库场景可传 `knowledge_recall` |
| `knowledgeBaseId` / `documentId` / `segmentId` | 否 | string | 溯源 |
| `imgKeywords` | 否 | string[] | **建议**与知识库一致，供 Resolve 使用 |
| `imgDescription` | 否 | string | 描述 |
| `tags` | 否 | string[] | 默认 `[]` |
| `domain` | 否 | string | 领域 |
| `description` | 否 | string | 顶层描述；未传时可用 `imgDescription` |

**行为**

- 若已存在相同 `sourceDetail.originalUrl`（即同一 `url`）的记录，**直接返回已有记录**（不重复下载）。
- 下载超时 **30s**；HTTP 非 2xx 或异常时失败。

**响应** `201`：元数据 + `url`

**错误**

- `400`：`{ "error": "url is required" }`
- `422`：`{ "error": "Failed to import image from URL" }`

---

### 4.8 `POST /api/images/resolve`

**Content-Type**: `application/json`

**Body**

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `sectionTitle` | 是 | string | 章节标题，参与分词 |
| `imageCount` | 是 | number | 需要返回的图片数量（整数） |
| `sectionId` | 否 | string | 蓝图组件 ID（当前匹配逻辑未使用，可预留） |
| `domain` | 否 | string | 传入时：仅保留 `visibility === "shared"` **或** `domain` 与传入值相等的记录 |
| `keywords` | 否 | string[] | 额外关键词，与标题拼接后分词 |

**响应** `200`

```json
{
  "resolved": [
    {
      "imageId": "uuid",
      "url": "/api/files/images/library/....jpg",
      "matchScore": 3,
      "source": "knowledge_recall"
    }
  ],
  "unresolved": 1
}
```

**错误**

- `400`：`sectionTitle` 缺失，或 `imageCount` 缺失/非 number

---

### 4.9 `POST /api/images/fetch`

**一站式图片获取**：给定关键词和数量，按四级优先级自动获取图片（图床匹配 → 知识库收割 → 联网搜图 → AI 生成），返回可直接使用的绝对 URL 列表。

**Content-Type**: `application/json`

**Body**

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `keywords` | 是 | string[] | 图片关键词数组（非空） |
| `count` | 是 | number | 需要多少张图（正整数） |
| `domain` | 否 | string | 限定领域 |
| `databaseId` | 否 | string | 知识库数据集 ID（传入时启用 Step 2 知识库收割） |
| `enableWebSearch` | 否 | boolean | 是否启用联网搜图（默认 `true`） |
| `autoGenerate` | 否 | boolean | 图片不足时是否 AI 生成补齐（默认 `true`） |
| `generateStyle` | 否 | string | AI 生成时的风格描述后缀 |
| `imageFilters` | 否 | object | Google Images 搜索过滤参数 |
| `imageFilters.size` | 否 | string | 图片尺寸：`large` / `medium` / `icon` 等 |
| `imageFilters.color` | 否 | string | 颜色：`color` / `transparent` / `black_and_white` 等 |
| `imageFilters.imageType` | 否 | string | 类型：`photo` / `clipart` / `line_drawing` / `gif` / `face` |
| `imageFilters.aspectRatio` | 否 | string | 宽高比：`square` / `tall` / `wide` / `panoramic` |
| `sessionId` | 否 | string | 关联会话 |

**四级执行流程**

1. **图床匹配** — 从全局图片库中按 imgKeywords/tags/description/generationPrompt 多维度评分匹配
2. **知识库收割** — 若提供 `databaseId` 且图片不足：用 keywords 搜知识库 → 提取 img_link → 下载入库 → 追加到结果
3. **联网搜图** — 若仍不足：使用 Google Images API（SearchApi.io `google_images` engine）进行专业图片搜索，支持尺寸/颜色/类型/宽高比等过滤参数；同时并行进行常规联网搜索提取页面内嵌图片 → 全部下载入库 → 追加到结果
4. **AI 生成** — 若仍不足且 `autoGenerate=true`：按关键词 AI 生成 → 入库 → 追加到结果

**响应** `200`

```json
{
  "images": [
    {
      "imageId": "uuid",
      "url": "http://192.168.31.45:3000/api/files/images/library/abc123.jpg",
      "source": "knowledge_recall",
      "matchScore": 4,
      "keywords": ["电路板", "焊接"]
    },
    {
      "imageId": "uuid",
      "url": "http://192.168.31.45:3000/api/files/images/library/def456.png",
      "source": "ai_generated",
      "matchScore": 0,
      "keywords": ["SMT贴片"]
    }
  ],
  "fromLibrary": 1,
  "fromKnowledgeBase": 0,
  "fromWebSearch": 0,
  "aiGenerated": 1,
  "failed": 0
}
```

所有图片 URL 均为绝对路径。每一级获取的图片都会**自动入库**，后续相同/相似关键词的调用可直接命中图床。

**错误**

- `400`：`keywords` 缺失/非数组/为空，或 `count` 缺失/非正数

## 5. Agent 工具 `image_library`

与 HTTP API 互补；在 `allowedTools` 中包含 `image_library` 时，由运行时注入租户/用户上下文。

| action | 作用 |
|--------|------|
| `fetch` | **一站式获取**：等同于调用 `POST /api/images/fetch`，四级管线自动获取图片 |
| `resolve` | 等同于调用服务层 `resolve`，参数放在 `context` 对象中 |
| `search` | 列表 + 可选 `query` 字符串过滤（匹配 keywords/tags/description/domain） |
| `get` | 按 `imageId` 取一条 |
| `save` | 等价于 `from-url` 入库，`sourceUrl` + 可选 `imgKeywords`（逗号分隔字符串）等 |

详细用法见各 Agent 的 `TOOLS.md`（如 `apps/document-generation/section-writer/TOOLS.md`）。

## 6. 环境变量

| 变量 | 说明 |
|------|------|
| `IMAGE_SERVE_BASE_URL` | 可选。设置后所有计算出的图片 `url` 使用该前缀（CDN/反代场景） |
| `NODE_ENV` | `production` 时未带 API Key 访问受保护接口会 401 |

---

## 7. 限制与后续规划

- 元数据单文件 JSON，规模大时需分片或迁移数据库（见方案设计文档）。
- Resolve 仅依赖 `imgKeywords`，未覆盖「仅 tags/description」的图片；Phase 3 可扩展溯源、向量检索、LLM 校验等。
- `POST /api/images/upload` 未从 HTTP 请求自动写入 `sessionId`（除非中间件扩展挂载 `request.sessionId`）。
