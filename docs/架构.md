# X-Pilot Agentic Server 架构文档

> 当前项目处于 **Demo/POC 阶段**：核心流程已跑通，配置偏示例，尚未达到生产级 SaaS 形态。

---

## 一、项目定位与阶段

| 表现 | 说明 |
|------|------|
| 配置偏示例 | `tenants.yaml` 使用示例 API Key |
| 依赖外部服务 | Dify、E2B、SearchApi 等均为可选/示例配置 |
| 认证宽松 | 无 tenants 时完全放行，开发环境无 API Key 也可用 |
| 单机部署 | 无集群、k8s 等生产级部署说明 |

## 四、核心模块说明

| 模块 | 职责 |
|------|------|
| **agent-registry** | 发现 apps/ 下 Agent、加载 prompts/config/skills、注册内置工具 |
| **agent-loader** | loadAgent()、discoverAgents()、loadAppModules() |
| **agent-runtime** | invokeAgent / streamAgent；路由到 pipeline 或 LLM 图 |
| **agent-graph** | LangGraph：perceive→think→act→observe→reflect |
| **pipeline-executor** | 拓扑 pipeline、fan-out 并行、自定义 handler |
| **sub-agent-manager** | spawn_sub_agent、spawn_parallel |
| **workspace** | 按 tenant/user/session 隔离的 artifacts、assets、logs |
| **auth** | tenants.yaml → API Key → tenantId / userId |
| **task-executor** | 异步任务队列（invoke-async、webhook） |
| **event-bus** | agent_started、progress、task_complete 等会话级事件 |

## 六、执行流程

### 6.1 启动流程

```
main() [src/index.ts]
  │
  ├─► loadConfig()                  # config/default.yaml + .env
  ├─► agentRegistry.initialize()
  │     ├─► registerBuiltInTools()
  │     ├─► registerBuiltInSkills()
  │     ├─► loadAppModules()        # apps/*/code/index.ts → routeRegistry
  │     └─► discoverAgents()        # 扫描 agent.config.yaml，loadAgent
  │
  ├─► HeartbeatRunner.start()
  └─► startServer()
```

### 6.2 请求处理流程

```
HTTP Request
  │
  ├─► auth 中间件 → API Key → tenantId, userId
  │
  ├─► 业务路由（如 document-generation/generate）
  │     ├─► 参数校验、buildInstruction
  │     └─► agentRuntime.invokeAgent() / streamAgent()
  │
  └─► agent-runtime
        │
        ├─ 有 pipeline 且 !skipPipeline
        │     └─► PipelineExecutor
        │           └─► 拓扑执行 steps → 调用子 Agent
        │
        └─ 无 pipeline 或 skipPipeline
              └─► buildGraph() → compiled.invoke() / .stream()
                    │
                    └─► LangGraph: perceive → think → act → observe → reflect
                          ├─► LLM 调用
                          ├─► 工具调用
                          └─► 写入 workspace（若有 sessionId）
```

### 6.3 document-generation 业务流

```
POST /api/business/document-generation/generate
  │
  ├─► getTenantId / getUserId（auth）
  ├─► normalizeRequest() → 校验 topic、template
  ├─► resolveTemplateContent() → 解析模板得 skeleton
  ├─► buildInstruction() → 构建发给 Agent 的自然语言指令
  │
  ├─► agentRuntime.invokeAgent("document-generation", instruction, {...})
  │     ├─► workspaceManager.ensureExists()
  │     ├─► buildGraph() → LangGraph
  │     ├─► compiled.invoke() → perceive→think→act→...
  │     └─► parseAgentOutput() → 按 outputFormat 解析
  │
  └─► 若 sessionId 存在
        └─► writeArtifact("artifacts/document.md", "artifacts/skeleton.json")
```

## 七、LLM 图（LangGraph）

```
START → perceive → think
         │
         ├─ 有 tool_calls → act（并行执行工具）→ observe
         │                    │
         │                    └─► 迭代次数 < max → think
         │                    └─► 3 次失败或达 max → reflect → END
         │
         └─ 无 tool_calls → reflect → END
```

## 八、数据存储

| 路径 | 内容 |
|------|------|
| `data/tenants/<tenantId>/agents/<agentName>/memory/` | 长期记忆、daily logs |
| `data/tenants/<tenantId>/users/<userId>/workspaces/<sessionId>/` | 会话工作区 |
| `workspaces/<sessionId>/artifacts/` | document.md、skeleton.json、script、code_array 等 |
| `workspaces/<sessionId>/assets/images/` | 生成的图片 |
| `workspaces/<sessionId>/logs/` | trace_*.json (执行追踪日志) |
| `apps/<agent>/memory/MEMORY.md` | 初始长期记忆 |

### 记忆分层

- **Working memory**：当次运行的短期状态
- **Daily log**：按日追加，定期 consolidation
- **Long-term memory**：合并后的 lessons / patterns，注入到 prompt

## 九、环境与启动

| 环境 | 命令 | 代码来源 |
|------|------|----------|
| 开发 | `npm run dev` | `src/`（tsx 实时编译） |
| 生产 | `npm run build` + `npm start` | `dist/`（需先 build） |
| 本地模拟生产 | `npm run start:prod` | 先 build 再 start |

环境判别：`process.env.NODE_ENV === "production"` 时启用严格认证（无 API Key 返回 401）。
